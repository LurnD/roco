# 洛克王国 AutoHotkey 辅助脚本

两个独立脚本，分别用于"采花循环"和"自动抓取"。两者基于 AutoHotkey v2.0 编写，带 GUI、支持后台/前台两种发送模式，配置自动持久化到同名 `.ini` 文件。

| 脚本 | 用途 | 启停热键 |
| --- | --- | --- |
| [roco_single_flower.ahk](roco_single_flower.ahk) | 单人采花循环（前置队伍设置 + 内循环 Tab/2/Esc/R/X） | `Ctrl + =` |
| [roco_auto_catch.ahk](roco_auto_catch.ahk) | 后台连续左键点击（用于抓宠等需要不停点击的场景） | `Ctrl + -` |

---

## 环境要求

- Windows
- 安装 [AutoHotkey v2.0](https://www.autohotkey.com/) 或更高版本
- 游戏进程名默认 `NRC-Win64-Shipping.exe`（洛克王国 UE 客户端）

## 运行方式

双击 `.ahk` 文件即可，脚本会自动请求管理员权限（用于发送按键到游戏窗口）。两个脚本可同时运行，热键互不冲突。

## roco_single_flower.ahk —— 单人采花循环

### 使用前准备

**队伍/背包配置（重要）**：
- **背包 1、3、4、5、6 号位放奇丽叶**
- **背包 2 号位放灵巧精灵**

脚本启动后会依次按 `1`→`3`→`4`→`5`→`6` 并伴随左键点击（"召唤"奇丽叶），随后按 `2` 切换到灵巧精灵作为主战，再进入 Tab/2/Esc/R/X 内循环采集。

### 使用步骤

1. 进入游戏，**传送到目标采集地点**
2. 双击运行脚本，GUI 会自动出现
3. 在"目标窗口"下拉里选择洛克王国窗口（找不到就点 `刷新`）
4. **点击 `初始化(↓×10)`** —— 这会连按 10 次下箭头来调整视角到合适角度
5. 视角调好后，**按 `Ctrl + =` 启动循环**（或点 `开始/停止` 按钮）
6. 想停下时再按一次 `Ctrl + =`

### GUI 字段说明

| 字段 | 说明 |
| --- | --- |
| 目标窗口 | 选择要操作的游戏窗口；多开时可选具体一个 |
| 前置按键序列 | 启动后先按一遍的按键（默认 `13456`），每个键后会跟一次客户区中心左键点击 |
| 循环按键序列 | 内循环的按键序列（默认 `Tab,2,Esc,R,X`），逗号分隔；每个键的等待时长在脚本内按键名预设（R 后等 ~10s，X 后等 50ms 等） |
| 循环次数最小/最大 | 内循环执行多少次后回到前置序列；填 `0` 表示**无限循环**（前置只跑一次） |
| 按键延迟 | 每次按键 down/up 之间的额外等待 (ms) |
| 走神概率% | 每次内循环开始时，按此百分比插入 10–20s 的随机停顿，模拟玩家走神 |
| 喘息概率% | 大循环结束时，按此百分比插入 5–10s 停顿；否则普通停顿 800–1200ms |
| 发送模式 | 后台 (PostMessage)：游戏可最小化，光标不会被抢走；前台：会激活窗口再用真实按键 |
| 窗口置顶 | GUI 是否始终在最上层；勾选状态会持久化 |

### 按钮

- **开始/停止 (Ctrl+=)**：启停循环（等价于全局热键）
- **发送按键测试**：单次发送一遍循环按键序列，用于验证按键能否进入游戏
- **初始化(↓×10)**：连按 10 次下箭头调整视角，不会自动启动循环
- **退出**：保存配置并退出脚本

### 后台 vs 前台

- 默认 **后台**：用 `PostMessage` 把按键直接送到窗口句柄，游戏窗口可以最小化或被遮挡
- 如果发现按键没反应（特别是 1/3/4/5/6 这类数字键），切到 **前台** 模式：脚本会先把游戏拉到前台再按真实按键。代价是按键期间不能操作其他窗口

## roco_auto_catch.ahk —— 后台自动连点

简单的左键自动连点器，用于抓宠等需要不停点同一个位置的场景。

### 使用步骤

1. 双击运行脚本，GUI 出现
2. 选择目标窗口
3. 选择 **点击位置**（见下文）
4. 按 `Ctrl + -` 启动连点；再按一次停止

### GUI 字段说明

| 字段 | 说明 |
| --- | --- |
| 目标窗口 | 同上 |
| 点击间隔最小/最大 | 两次点击之间的随机间隔 (ms)，默认 800–1200 |
| 按键停留最小/最大 | 鼠标按下到松开的时长 (ms)，默认 30–60，模拟真人点按 |
| 发送模式 | 同 single_flower 脚本：后台 / 前台 |
| 点击位置 | 三选一：窗口客户区中心 / 自定义坐标 / 跟随鼠标位置 |
| 自定义 X / Y | 当模式为 "自定义坐标" 时使用，相对游戏窗口客户区左上角的像素 |
| 窗口置顶 | 同上 |

### 三种点击位置模式

- **窗口客户区中心**：每次点击都按当前窗口尺寸算中心，窗口缩放也跟着变
- **自定义坐标**：固定点同一个相对位置，最适合点固定的按钮
- **跟随鼠标位置**：实时换算屏幕鼠标位置到客户区相对坐标 —— 等价于 1.x 老脚本"点击当前光标处"的行为

### 按钮

- **开始/停止 (Ctrl+-)**：启停连点
- **测试单击**：发送一次点击，用于验证位置和模式
- **记录当前鼠标**：把当前鼠标在游戏窗口的相对坐标抓到自定义 X/Y 字段，并自动切到"自定义坐标"模式 —— 方便定位按钮：把鼠标停在想点的位置上，按这个按钮即可
- **退出**：保存配置并退出

## 配置持久化

每个脚本的配置会自动保存到同目录下的同名 `.ini` 文件：

- `roco_single_flower.ini`
- `roco_auto_catch.ini`

修改 GUI 里任意字段并点开始/测试/初始化等按钮，或退出脚本时，配置会自动写入。下次启动会读取上次的设置。

## 常见问题

**Q: 按 `Ctrl + =` 浏览器会放大怎么办？**  
A: AHK 的全局热键优先级高于浏览器，但只在脚本运行期间拦截。脚本未运行时浏览器行为正常。

**Q: 后台模式下按键完全没反应？**  
A: 把"发送模式"切到"前台"。洛克王国是 UE 引擎，部分按键可能只接受真实输入。

**Q: 想同时跑两个脚本？**  
A: 可以。两个脚本热键不同（`Ctrl+=` 和 `Ctrl+-`），ini 配置也独立。

**Q: 进程名不对怎么办？**  
A: 进程名硬编码在脚本顶部（`GetDefaultSettings()` 里的 `targetExe`），改完删掉对应 ini 重启即可。
